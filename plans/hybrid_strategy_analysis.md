## 🚨 Критические проблемы

### 1. Нерентабельные параметры для RANGING режима

**Конфигурация** (`config/trading.json:72-77`):
```json
"RANGING": {
  "min_score": 5,
  "sl_multiplier": 1,      // ← Слишком узкий SL
  "tp_multiplier": 2,      // ← Слишком узкий TP
  "position_size_factor": 0.8
}
```

**Проблема**:
- При ATR = 0.2971 и цене ~613:
  - SL = 613 - (0.2971 × 1.0 × 0.94) = 612.72 → **-0.05%**
  - TP = 613 + (0.2971 × 2.0 × 1.08) = 613.44 → **+0.07%**
- Комиссия round trip = 0.200% (0.05% taker × 2)
- **Даже при достижении TP, сделка убыточна**: 0.07% - 0.200% = **-0.13%**

### 2. Неправильная формула валидации

**Код** (`src/core/risk_manager.py:236-240`):
```python
round_trip_fee_pct = TRADING_FEE_TAKER * 2.0  # 0.100%
effective_rr = (reward_pct - round_trip_fee_pct) / (risk_pct + round_trip_fee_pct)
```

**Проблема**: Формула корректна, но параметры SL/TP делают её неработоспособной:
- reward_pct = 0.07%
- risk_pct = 0.05%
- effective_rr = (0.07 - 0.200) / (0.05 + 0.200) = -0.13 / 0.25 = **-0.52**

### 3. Низкое качество сигнала

- Quality = 0.28 (очень низкое)
- Это приводит к:
  - SL_adj = 1.0 - (0.28 × 0.2) = 0.94 (SL ближе к цене)
  - TP_adj = 1.0 + (0.28 × 0.3) = 1.08 (TP дальше от цены)
- **Эффект**: SL становится ещё ближе, TP дальше, но разница всё ещё недостаточна

### 4. Низкая волатильность рынка

- ATR = 0.2971 при цене 613 → ATR ratio = 0.048% (очень низкий)
- В режиме RANGING с низкой волатильностью:
  - SL_mult = 1.0 слишком мал
  - TP_mult = 2.0 слишком мал
  - Не покрывает комиссии

## 🎯 Рекомендации по исправлению

### Приоритет 1: Увеличить TP_mult для RANGING режима

**Текущее значение**: `tp_multiplier: 2`
**Рекомендуемое значение**: `tp_multiplier: 3.5` (как в TRENDING)

**Обоснование**:
- При TP_mult = 3.5:
  - TP = 613 + (0.2971 × 3.5 × 1.08) = 614.12 → **+0.18%**
  - Net R/R = (0.18 - 0.200) / (0.05 + 0.200) = -0.02 / 0.25 = **-0.08** (всё ещё плохо)
- Нужно ещё больше: `tp_multiplier: 5` или выше

### Приоритет 2: Увеличить SL_mult для RANGING режима

**Текущее значение**: `sl_multiplier: 1`
**Рекомендуемое значение**: `sl_multiplier: 1.5` (как в TRENDING)

**Обоснование**:
- Увеличит risk_pct, что улучшит соотношение
- Даст больше пространства для манёвра

### Приоритет 3: Добавить проверку на минимальный ATR ratio

**Проблема**: При очень низком ATR ratio (< 0.1%) сделки нерентабельны
**Решение**: Добавить фильтр в `signal_generator.py`:
```python
if atr_ratio < 0.1:  # Минимум 0.1% движения
    return "HOLD", 0, 0.0
```

### Приоритет 4: Снизить min_risk_reward_ratio для RANGING

**Текущее значение**: `min_risk_reward_ratio: 1.2`
**Рекомендуемое значение**: `min_risk_reward_ratio: 0.8` для RANGING режима

**Обоснование**:
- В режиме RANGING с низкой волатильностью难以 достичь R/R > 1.2
- Более реалистичное требование: 0.8

### Приоритет 5: Улучшить качество сигнала

**Проблема**: Quality = 0.28 слишком низкое
**Решение**:
- Увеличить `min_score_for_signal` с 4 до 6 в HYBRID конфиге
- Добавить дополнительные фильтры (объём, волатильность)
