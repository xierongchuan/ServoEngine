# Стратегия MACDX

## Общее описание

MACDX — это детерминистическая (без AI) торговая стратегия, основанная на пересечениях MACD с подтверждениями от других индикаторов.

**Тип:** Trend-following (следование за трендом)
**Таймфрейм:** 1 минута (с анализом 1 дня)
**Без AI:** Да (полностью математический расчёт)

---

## 1. Индикатор MACD (Основной сигнал)

### Параметры
- Fast EMA: 12
- Slow EMA: 26
- Signal Line: 9

### Логика пересечения

**Бычье пересечение (BUY):**
```
macd_hist_prev <= 0 И macd_hist > 0
```
Гистограмма переходит из отрицательной в положительную (пересекает ноль).

**Медвежье пересечение (SELL):**
```
macd_hist_prev >= 0 И macd_hist < 0
```
Гистограмма переходит из положительной в отрицательную.

**ВАЖНО:** Пересечение определяется только по СМЕНЕ ЗНАКА гистограммы, а не по росту значения!

---

## 2. Подтверждения (Confirmations)

Стратегия требует минимум 3-5 подтверждений для открытия сделки.

### Таблица весов

| # | Индикатор | Вес | Описание |
|---|-----------|-----|----------|
| 1 | MACD Crossover | +2 | Основной сигнал (обязательно) |
| 2 | RSI Zone | +2 | RSI в зоне комфорта |
| 3 | EMA Alignment | +2 | EMA 9 > EMA 21 (или наоборот) |
| 4 | Not Sideways | +1 | Рынок в тренде (не боковик) |
| 5 | No Exhaustion | +1 | Нет дивергенции RSI |
| 6 | Volume | +1 | Достаточный объём |

**Максимальный балл:** 9
**Минимум для сигнала:** 4 бала
**Минимум подтверждений:** 3

---

## 3. Подробное описание подтверждений

### 3.1 RSI Zone (+2)

**Для LONG:**
- RSI должен быть между 25 и 65
- `rsi_long_min = 25`
- `rsi_long_max = 65`

**Для SHORT:**
- RSI должен быть между 35 и 75
- `rsi_short_min = 35`
- `rsi_short_max = 75`

**Защита от экстремумов:**
- При RSI < 30 (экстремальная перепроданность) — блокируется SELL
- При RSI > 70 (экстремальная перекупленность) — блокируется BUY

### 3.2 EMA Alignment (+2)

- **LONG:** EMA 9 > EMA 21
- **SHORT:** EMA 9 < EMA 21

**Штраф за контр-тренд:**
- Если EMA направлен против MACD пересечения — добавляется предупреждение "counter-trend"

### 3.3 Not Sideways (+1)

Определение бокового рынка:

```python
bb_width = (bb_upper - bb_lower) / bb_middle * 100
is_sideways = (bb_width < 0.5%) AND (adx < 20)
```

**Условие для подтверждения:**
- BB width > 0.5% (широкие полосы) ИЛИ
- ADX > 20 (есть тренд)

### 3.4 No Exhaustion / Дивергенция (+1)

Проверка на RSI дивергенцию:

**Бычья дивергенция (для LONG):**
- Цена делает более низкие минимумы
- RSI делает более высокие минимумы
- Это сигнал истощения медведей → блокируем SELL

**Медвежья дивергенция (для SHORT):**
- Цена делает более высокие максимумы
- RSI делает более низкие максимумы
- Это сигнал истощения быков → блокируем BUY

**Штраф:** -1 балл при наличии дивергенции

### 3.5 Volume (+1)

- **Подтверждение:** volume_ratio >= 0.8x
- Можно отключить через `enable_volume_filter: false`

---

## 4. Фильтры (Блокировка сигнала)

### 4.1 Волатильность (Hard Filter)

Если `atr_ratio < 0.3` — сигнал блокируется (низкая волатильность).

### 4.2 Объём (Hard Filter)

Если `volume_ratio < 0.5` и фильтр включён — сигнал блокируется.

### 4.3 Фильтр последовательных свечей

Блокировка LONG:
- 4+ красных свечей (STRONG_DOWN) + медвежий MACD momentum

Блокировка SHORT:
- 4+ зелёных свечей (STRONG_UP) + бычий MACD momentum

### 4.4 Контр-тренд защита

Если EMA движется против направления MACD пересечения с разницей > 1% — сигнал блокируется.

## 6. Расчёт SL и TP (Детерминированный)

**Stop Loss:**
```
SL = цена × (1 - atr × atr_sl_mult / 100)
```

**Take Profit:**
```
TP = цена × (1 + atr × atr_tp_mult / 100)
```

**Пример (BTC = $70,000, ATR = $100):**
- SL = 70,000 × (1 - 100 × 1.5 / 100) = 70,000 × 0.985 = $68,950
- TP = 70,000 × (1 + 100 × 3.0 / 100) = 70,000 × 1.03 = $72,100
- SL% = 1.5%, TP% = 3.0%
- R/R = 3.0 / 1.5 = 2.0

---

## 7. Risk Validation (Опционально)

Это дополнительная проверка, которую можно отключить. По умолчанию ВЫКЛЮЧЕНА для MACDX.

Проверяет, что Net R/R >= 1.2 после учёта комиссий:
- Gross R/R = TP% / SL%
- Net R/R = (TP% - 0.2%) / (SL% + 0.2%)

**Зачем нужно:** Защищает от сделок с очень маленьким SL/TP, где комиссия съедает прибыль.

**Пример проблемы:**
- ATR = $10 (низкая волатильность)
- SL% = 0.15%, TP% = 0.30%
- Gross R/R = 2.0 (хорошо)
- Net R/R = (0.30 - 0.2) / (0.15 + 0.2) = 0.28 (ПЛОХО!)

**Как отключить:** Установите `min_risk_reward_ratio: 0` в конфиге.

## 8. Логика валидации риска

Перед открытием позиции проверяется соотношение риск/прибыль:

1. **Gross R/R** = TP% / SL%
2. **Net R/R** = (TP% - 0.2%) / (SL% + 0.2%)
   - 0.2% = комиссия туда-обратно (0.1% × 2)

**Условие:** Net R/R >= 1.2

Если TP слишком близок к цене входа (менее 0.2%), Net R/R будет 0 или отрицательным → сделка отклоняется.
