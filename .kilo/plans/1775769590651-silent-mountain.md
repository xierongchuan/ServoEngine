# План создания бэктеста для первого символа

## Цель
Создать модуль бэктеста, который симулирует торговлю на исторических данных из `data/prices/` только для первого символа из списка в конфигурации. Бэктест должен рассчитывать прибыльность, комиссии и позиции без реального взаимодействия с биржей, обеспечивая быструю итерацию без задержек. Интеграция должна быть аккуратной, следуя архитектуре проекта, с возможностью отдельного запуска через скрипт в `scripts/`.

## Текущая архитектура (кратко)
- **Конфигурации**: Иерархическая загрузка из `config/`.
- **Символы**: Список активных в `config/active.json` под `"symbols":{"bingx":[...]}"`.
- **Данные цен**: Хранятся в `data/prices/{SYMBOL}.json` как массив klines (snapshotTimeUTC, open/high/low/close/volume).
- **Торговля**: Через BingX клиент с кэшированием позиций (5с), баланса (10с), комиссий (1ч). Позиции как dict с unrealized_pnl, комиссии maker/taker (0.02%/0.05%).
- **Процессы**: Каждый символ в отдельном OS процессе, изоляция.
- **Стратегии**: Определены в `config/strategies/`, с AI для HYBRID.

## Определение первого символа
Первый символ берется из `config/active.json["symbols"]["bingx"][0]`, нормализуется к "BTC-USDT" для внутренних расчетов. Если список пуст или disabled_symbols содержит его, пропустить.

## Архитектура бэктеста
- **Модуль**: Новый `src/backtest/` с классом `BacktestEngine`, наследующим паттерны как `MarketRegimeDetector` (singleton per процесс).
- **Компоненты**:
  - `BacktestSimulator`: Симулирует позиции, ордера, SL/TP.
  - `CommissionCalculator`: Рассчитывает комиссии на основе конфигурации (maker/taker).
  - `PnLTracker`: Вычисляет realized/unrealized P&L.
  - `DataLoader`: Загружает и итерирует klines из `data/prices/`.
  - `SignalGenerator`: Генерирует сигналы для детерминированных стратегий (MACDX) на основе технических индикаторов.
- **Итерация**: Цикл по klines без задержек, симуляция на каждом шаге.
- **Метрики**: Win rate, Sharpe ratio, max drawdown, total P&L, commission impact.
- **Интеграция**: Использовать существующие классы (BingXClient для комиссий, но без API), логирование на русском с эмодзи.

## Шаги реализации
1. **Создать структуру модуля `src/backtest/`**:
   - `__init__.py` с импортами.
   - `engine.py` для `BacktestEngine`.
   - `simulator.py` для симуляции.
   - `metrics.py` для расчетов метрик.
   - `signals.py` для `SignalGenerator`.

2. **Реализовать DataLoader**:
   - Получить данные из биржи по timeframe из конфигурации (например, 15m для MACDX) используя BingXClient.get_klines(), сохранить в `data/prices/{SYMBOL}.json` если нужно.
   - Загрузить JSON из `data/prices/{SYMBOL}.json`.
   - Итерировать по klines в порядке времени.

3. **Реализовать симуляцию позиций**:
   - Моделировать создание/закрытие позиций на основе сигналов стратегии.
   - Рассчитывать entry_price, size, leverage из конфигурации.
   - Мониторить SL/TP на каждом шаге.

4. **Добавить расчет комиссий**:
   - Использовать ставки из `get_commission_rates()`, но симулировать без API.
   - Применять maker для лимитных, taker для маркет.

5. **Интегрировать с конфигурацией**:
   - Загружать активную конфигурацию для символа.
   - Поддерживать hot-reload симуляции.

6. **Создать скрипт запуска `scripts/run_backtest.py`**:
   - Парсить аргументы: символ (по умолчанию первый), период, стратегия.
   - Запускать BacktestEngine, выводить метрики.

7. **Добавить логирование и отчеты**:
   - Логи в `data/logs/backtest_{SYMBOL}.log`.
   - Отчет в `data/backtest_result.json` с trades, P&L, метриками.

## Интеграция и лучшие практики
- **Не ломать проект**: Бэктест изолирован, не влияет на реальную торговлю. Использовать паттерны проекта (singletons, логирование).
- **Тестирование**: Запускать в контейнере как указано в AGENTS.md. Покрыть тестами в `tests/test_backtest.py`.
- **Безопасность**: Не логировать секреты, следовать правилам.
- **Документация**: Обновить CLAUDE.md с описанием бэктеста.

## Уточнения на основе ответов
- **Период данных**: Получать все данные из биржи по timeframe из конфигурации (например, свечи за неделю).
- **AI сигналы**: Не интегрировать, фокус на детерминированных стратегиях как MACDX.
- **Метрики**: Стандартные для бэктестов (win rate, total P&L, commissions, max drawdown, Sharpe ratio) - торговля как обычно, ускоренное время.
- **Вывод результатов**: data/backtest_result.json.
