# План реализации: Оптимизация выходов из позиций в MACDX стратегии

## Цель
Реализовать точные и надёжные алгоритмы выхода из позиций для максимизации прибыли:
1. Выход в середине MACD тренда (когда импульс начинает ослабевать)
2. Быстрое закрытие на удачных пампах (секунды)

## Анализ текущей логики выхода (MACDX)

### Существующие сигналы закрытия:
- **MACD разворот**: Гистограмма меняет знак (hist < 0 для лонгов)
- **RSI экстремумы**: >80 для лонгов, <20 для шортов
- **Прибыль с RSI**: +3% прибыли + экстремальный RSI
- **Отсутствие динамики**: Только статические правила

### Проблемы текущей системы:
- Выход только при полном развороте тренда (упущенная прибыль)
- Нет быстрого закрытия на пампах
- Нет масштабирования позиций

## Предлагаемая система выходов

### 1. Выход в середине MACD тренда

#### Алгоритм ослабления импульса:
```python
# Отслеживание пика гистограммы
class MacdTracker:
    def __init__(self):
        self.peak_hist = 0
        self.peak_time = 0

    def update_peak(self, hist):
        if hist > self.peak_hist:
            self.peak_hist = hist
            self.peak_time = time.time()

    def get_weakening_ratio(self, current_hist):
        return current_hist / self.peak_hist if self.peak_hist > 0 else 1.0
```

#### Динамические уровни выхода:
- **Ранний выход** (70% от пика): Для агрессивных настроек
- **Оптимальный выход** (50% от пика): Баланс прибыли/риска
- **Поздний выход** (30% от пика): Для удержания тренда

### 2. Быстрое закрытие на пампах

#### Временной фильтр с подтверждениями:
```python
entry_time = position.entry_timestamp
time_in_trade = time.time() - entry_time

# Быстрое закрытие: 10-30 секунд + минимальная прибыль
if time_in_trade < 30 and pnl_pct >= 1.0:
    if volume_recent > volume_avg * 2.0:  # Объём подтверждает памп
        return {"should_close": True, "reason": "Quick pump with volume", "urgency": "critical"}

# Сверхбыстрое закрытие: 5 секунд + высокая прибыль
if time_in_trade < 5 and pnl_pct >= 2.0:
    return {"should_close": True, "reason": "Instant pump profit", "urgency": "critical"}
```

#### Импульсные индикаторы:
- **RSI памп**: RSI > 75 после спокойного периода
- **Ценовой импульс**: Δцена > 1.5% за 15 секунд
- **Объёмный взрыв**: volume_ratio > 4.0 в последних 3 свечах

### 3. Масштабирование выхода (Scale-out)

#### Многоуровневое закрытие:
```python
# Первый уровень: 25% позиции при +0.8% (быстрая прибыль)
if pnl_pct >= 0.8:
    partial_close(0.25, "Early profit scaling")

# Второй уровень: 50% при ослаблении тренда
if macd_weakening_ratio < 0.6:
    partial_close(0.5, "Momentum weakening")

# Финальный уровень: оставшиеся при полном развороте
if macd_hist < 0:
    partial_close(1.0, "Full trend reversal")
```

#### Динамический размер долей:
- При сильном тренде: Меньше долей на ранних выходах
- При слабом тренде: Больше долей на ранних выходах

### 4. Защитные и адаптивные механизмы

#### Адаптивные стоп-лоссы:
```python
# ATR-based trailing stop
stop_distance = atr * 1.5
if current_price > entry_price + stop_distance:
    trailing_stop = max(trailing_stop, current_price - stop_distance)
```

#### Анти-паника фильтры:
- **Минимальное время удержания**: Не закрывать раньше 3 секунд
- **Подтверждение тренда**: Требовать 2+ свечи подтверждения
- **Объёмный фильтр**: Игнорировать пампы без объёма

#### Риск-менеджмент:
- **Максимальная просадка**: Закрыть если убыток > 2%
- **Временной лимит**: Закрыть через 10 минут независимо от прибыли
- **Корреляция активов**: Не держать все позиции в одном направлении
