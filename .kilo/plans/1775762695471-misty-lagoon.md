# План реализации: Оптимизация выходов из позиций в MACDX стратегии

## Цель
Реализовать точные и надёжные алгоритмы выхода из позиций для максимизации прибыли:
1. Выход на пике прибыли MACD тренда (когда импульс начинает ослабевать)
2. Фиксация прибыли на сильных импульсных свечах (минуты, не секунды)
3. Экстренные выходы при развороте тренда

## Анализ текущей логики выхода (MACDX)

### Существующие сигналы закрытия (`src/core/signals/macdx.py:should_close`):
- **MACD разворот + прибыль**: Гистограмма < 0 для лонгов + PnL >= 0.5% → закрытие (medium urgency)
- **MACD разворот + убыток**: Гистограмма < 0 для лонгов + PnL < -1.0% → закрытие (high urgency)
- **RSI экстремумы**: RSI > 80 для лонгов, RSI < 20 для шортов → безусловное закрытие
- **Прибыль с RSI**: PnL >= +3% при RSI > 70 (лонг) или RSI < 30 (шорт)
- **Зеркальная логика для шортов**: Аналогичные правила с hist > 0

### Сильные стороны текущей системы:
- Комбинированные условия (MACD + PnL), а не простые пороги
- Разная urgency для разных ситуаций
- Поддержка как dict так и object позиций

### Проблемы текущей системы:
- Выход только при полном развороте гистограммы (упущенная прибыль на ослаблении тренда)
- Нет отслеживания пиковой прибыли и просадки от пика
- Нет учёта времени в позиции
- Нет масштабирования порогов по волатильности (ATR)
- Порог -1% потерь статический, не учитывает плечо и ATR
- Нет интеграции с рыночным режимом (regime) при принятии решений о выходе

## Предлагаемая система выходов

### 1. Выход на ослаблении импульса MACD (пиковая прибыль)

#### Алгоритм отслеживания пика гистограммы:
```python
class MacdMomentumTracker:
    """Отслеживает пик гистограммы MACD для определения ослабления тренда.
    
    Работает на уровне свечей (1h), не в реальном времени.
    Обновляется при каждом цикле (каждые 60 секунд).
    """
    def __init__(self):
        self.peak_hist = 0           # Абсолютный пик гистограммы с момента входа
        self.peak_candle_idx = 0     # Индекс свечи с пиком
        self.candles_since_entry = 0 # Количество свечей с момента входа

    def update(self, hist: float):
        """Обновить при каждом новом значении гистограммы."""
        self.candles_since_entry += 1
        abs_hist = abs(hist)
        if abs_hist > abs(self.peak_hist):
            self.peak_hist = hist
            self.peak_candle_idx = self.candles_since_entry

    def get_weakening_ratio(self, current_hist: float) -> float:
        """Отношение текущей гистограммы к пику. 1.0 = на пике, 0.0 = полный разворот."""
        if self.peak_hist == 0:
            return 1.0
        # Для лонгов: peak_hist > 0, ratio падает при уменьшении hist
        # Для шортов: peak_hist < 0, ratio падает при увеличении hist (к 0)
        if self.peak_hist > 0:
            return max(0, current_hist / self.peak_hist)
        else:
            return max(0, current_hist / self.peak_hist) if current_hist < 0 else 0

    def candles_since_peak(self) -> int:
        """Количество свечей после пика."""
        return self.candles_since_entry - self.peak_candle_idx
```

#### Динамические уровни выхода на основе ослабления:
```python
# Контекст: 1h таймфрейм, обновление каждые 60 секунд
# Мы наблюдаем ослабление не в секундах, а в свечах

tracker = position_context.get("macd_tracker")
weakening = tracker.get_weakening_ratio(macd_hist)
candles_after_peak = tracker.candles_since_peak()

# Уровень 1: Консервативный выход (50% от пика + 3+ свечей после пика)
# Пример: hist пик=0.8, текущий=0.4 → weakening=0.5
if weakening <= 0.5 and candles_after_peak >= 3 and pnl_pct >= 0.5:
    return {"should_close": True, "reason": f"MACD momentum weakened to {weakening:.0%} (peak exit)", "urgency": "medium"}

# Уровень 2: Пересечение сигнальной линии (запасной сигнал)
if position_long and macd_line < macd_signal and prev_macd_line >= prev_macd_signal:
    return {"should_close": True, "reason": "MACD bearish signal crossover", "urgency": "medium"}
```

**Важно**: Ослабление до 70% слишком агрессивно для 1h свечей — нормальные откаты тренда легко достигают 60-70% от пика. Порог 50% + минимум 3 свечи после пика — баланс между фиксацией прибыли и удержанием тренда.

### 2. Фиксация прибыли на импульсных свечах

#### Контекст: 1h таймфрейм, цикл 60 секунд
MACDX работает на часовых свечах с циклом 60 секунд. «Пампы» в контексте MACDX — это сильные часовые свечи (>2% тело), а не секундные движения.

```python
# Импульсная свеча: тело > 2 * ATR (аномально большая свеча)
candle_body = abs(close - open_price)
is_impulse_candle = candle_body > atr * 2.0

# Фиксация на импульсной свече с прибылью
if is_impulse_candle and pnl_pct >= 1.5:
    return {"should_close": True, "reason": f"Impulse candle +{pnl_pct:.1f}% (body={candle_body/atr:.1f}x ATR)", "urgency": "high"}

# Объёмное подтверждение импульса
if is_impulse_candle and volume_ratio > 2.5 and pnl_pct >= 1.0:
    return {"should_close": True, "reason": f"Volume impulse +{pnl_pct:.1f}% (vol={volume_ratio:.1f}x)", "urgency": "high"}
```

**Почему не в секундах**: MACDX анализирует закрытые 1h свечи. Промежуточные тики внутри свечи для него не релевантны — сигналы рассчитываются по закрытым свечам. Мониторинг на уровне секунд — это территория SCALP стратегии, не MACDX.

### 3. Trailing profit (защита накопленной прибыли)

Вместо масштабирования позиций (что добавляет сложность и комиссии), используем trailing profit — динамическую защиту прибыли от пика PnL.

```python
# Отслеживание пика прибыли
peak_pnl = position_context.get("peak_pnl", 0)
peak_pnl = max(peak_pnl, pnl_pct)
position_context["peak_pnl"] = peak_pnl

# Trailing profit: закрываем если прибыль упала более чем на 40% от пика
# Пример: пик +5%, текущая +3% → просадка от пика 40% → закрытие
if peak_pnl >= 2.0:  # Активируется только при значительной прибыли
    drawdown_from_peak = (peak_pnl - pnl_pct) / peak_pnl
    
    # Адаптивный порог: чем больше прибыль, тем строже защита
    if peak_pnl >= 5.0:
        max_drawdown = 0.30  # Защищаем 70% прибыли при пике 5%+
    elif peak_pnl >= 3.0:
        max_drawdown = 0.40  # Защищаем 60% прибыли при пике 3-5%
    else:
        max_drawdown = 0.50  # Защищаем 50% прибыли при пике 2-3%
    
    if drawdown_from_peak >= max_drawdown:
        return {
            "should_close": True,
            "reason": f"Trailing profit: peak +{peak_pnl:.1f}%, now +{pnl_pct:.1f}% (drawdown {drawdown_from_peak:.0%})",
            "urgency": "medium"
        }
```

### 4. Защитные и адаптивные механизмы

#### ATR-адаптивный стоп-лосс (trailing):
```python
# ATR-based trailing stop — привязан к пиковой цене, не к entry
atr_mult = config.get("preset", {}).get("atr_sl_mult", 1.5)
stop_distance = atr * atr_mult  # 1.5 * ATR по умолчанию

# Активируется при прибыли >= 1 ATR
activation_threshold = atr
if pos_type == "BUY":
    if current_price > entry_price + activation_threshold:
        # Trailing от максимальной цены
        peak_price = position_context.get("peak_price", current_price)
        peak_price = max(peak_price, current_price)
        position_context["peak_price"] = peak_price
        trailing_stop = peak_price - stop_distance
        
        if current_price <= trailing_stop:
            return {"should_close": True, "reason": f"ATR trailing stop hit ({trailing_stop:.2f})", "urgency": "high"}
```

#### Экстренные выходы при развороте тренда:
```python
def should_emergency_close(position, analysis, macd_hist, macd_hist_prev):
    """Экстренные выходы — только при подтверждённых разворотах."""
    pos_type = "BUY" if position.is_long else "SELL"
    
    # Защита: не закрывать если позиция соответствует текущему MACD тренду
    position_matches_trend = (
        (pos_type == "BUY" and macd_hist > 0) or
        (pos_type == "SELL" and macd_hist < 0)
    )
    if position_matches_trend:
        return {"should_close": False, "reason": "Position matches MACD trend"}
    
    # 1. Резкий разворот гистограммы (смена знака + значительная величина)
    # Порог 0.1 слишком мал для 1h — используем ATR-масштабированный порог
    atr = analysis.get("atr", 0)
    macd_reversal_threshold = atr * 0.001  # Масштабирован к ATR
    
    if pos_type == "BUY" and macd_hist_prev > 0 and macd_hist < -macd_reversal_threshold:
        return {"should_close": True, "reason": "MACD instant reversal (bearish)", "urgency": "critical"}
    if pos_type == "SELL" and macd_hist_prev < 0 and macd_hist > macd_reversal_threshold:
        return {"should_close": True, "reason": "MACD instant reversal (bullish)", "urgency": "critical"}
    
    # 2. Убыток превышает 2 * ATR (динамический макс. убыток)
    entry_price = float(position.entry_price)
    current_price = analysis.get("current_price", 0)
    if pos_type == "BUY":
        loss_in_atr = (entry_price - current_price) / atr if atr > 0 else 0
    else:
        loss_in_atr = (current_price - entry_price) / atr if atr > 0 else 0
    
    if loss_in_atr > 2.0:
        return {"should_close": True, "reason": f"Max loss exceeded ({loss_in_atr:.1f}x ATR)", "urgency": "critical"}
    
    return {"should_close": False, "reason": "No emergency conditions"}
```

#### Учёт рыночного режима при выходе:
```python
# Режим уже рассчитывается в MacdxPipeline.generate_command()
# Используем его для адаптации порогов выхода

regime = analysis.get("regime", {}).get("regime", "UNKNOWN")

if regime == "TRENDING":
    # В тренде — даём позиции больше пространства
    weakening_threshold = 0.40    # Закрываем при 40% от пика (агрессивное удержание)
    trailing_drawdown = 0.50      # Допускаем 50% просадки от пика прибыли
    min_profit_for_impulse = 2.0  # Импульсные выходы только при 2%+ прибыли
elif regime == "RANGING":
    # В боковике — фиксируем быстрее
    weakening_threshold = 0.60    # Закрываем при 60% от пика (быстрая фиксация)
    trailing_drawdown = 0.35      # Допускаем только 35% просадки
    min_profit_for_impulse = 0.8  # Импульсные выходы уже при 0.8%
elif regime == "VOLATILE":
    # В волатильности — защита прежде всего
    weakening_threshold = 0.55
    trailing_drawdown = 0.30
    min_profit_for_impulse = 1.0
else:  # TRANSITIONAL / UNKNOWN
    weakening_threshold = 0.50
    trailing_drawdown = 0.40
    min_profit_for_impulse = 1.5
```

#### Анти-паника фильтры (калиброваны под 1h):
- **Минимальное количество свечей**: Не закрывать по ослаблению раньше 2-х полных свечей после входа (2 часа)
- **Подтверждение тренда**: Требовать что ослабление продолжается 2+ свечи подряд (не просто откат одной свечи)
- **Не закрывать в плюсе при сильном ADX**: Если ADX > 30 и позиция прибыльна — удерживать (сильный тренд)

#### Риск-менеджмент:
- **Максимальная просадка**: ATR-масштабированная (2x ATR), не фиксированные проценты
- **Временной лимит**: 48 часов для 1h стратегии (80 свечей) — разумный горизонт, а не 10 минут
- **Максимальный стоп**: Плечо × ATR × SL_mult — учитывает конфигурацию левеража (6x)

### Расширение should_close с приоритетами:
```python
def should_close(self, analysis: Dict, position: Any, **kwargs) -> Dict:
    """
    Приоритизация выходов:
    1. Экстренные выходы (critical) — немедленное закрытие
    2. Trailing profit / ATR trailing stop (high) — защита прибыли
    3. Ослабление MACD импульса (medium) — оптимальный выход на пике
    4. Импульсные свечи + прибыль (medium) — фиксация на волатильности
    5. Текущие правила (RSI экстремумы, MACD разворот + PnL)
    """
    if not position:
        return {"should_close": False, "reason": "No position", "urgency": "low"}

    # ... existing position parsing ...
    
    position_context = kwargs.get("position_context", {})
    regime = analysis.get("regime", {})

    # 1. ЭКСТРЕННЫЕ ВЫХОДЫ — максимальный приоритет
    emergency = self._check_emergency_exit(analysis, position, pos_type, pnl_pct)
    if emergency.get("should_close"):
        return emergency

    # 2. TRAILING PROFIT — защита накопленной прибыли
    trailing = self._check_trailing_profit(pnl_pct, position_context, regime)
    if trailing.get("should_close"):
        return trailing

    # 3. ОСЛАБЛЕНИЕ MACD ИМПУЛЬСА — выход на пике тренда
    momentum = self._check_macd_weakening(analysis, position_context, pnl_pct, regime)
    if momentum.get("should_close"):
        return momentum

    # 4. ИМПУЛЬСНЫЕ СВЕЧИ — фиксация на волатильности
    impulse = self._check_impulse_exit(analysis, pnl_pct, regime)
    if impulse.get("should_close"):
        return impulse

    # 5. СУЩЕСТВУЮЩИЕ ПРАВИЛА (MACD reversal + PnL, RSI extreme, TP)
    return self._check_standard_exits(analysis, pos_type, pnl_pct)
```

### Метрики сравнения:
- **Win rate**: Текущий → ожидаемый +3-5% за счёт выхода ближе к пику
- **Average profit per trade**: Текущий → ожидаемый +10-15% за счёт trailing profit
- **Max drawdown**: Текущий → ожидаемый -15-20% за счёт ATR trailing stop
- **Sharpe ratio**: Ключевая метрика — должен расти
- **Average hold time**: Ожидаемо сократится с N часов до оптимума

## Технические вызовы и решения

### Сохранение exit_context между циклами:
- **Проблема**: MacdxSignalGenerator создаётся заново каждый цикл (`gen = MacdxSignalGenerator(settings)`)
- **Решение**: Хранить exit_context в active_trades.json через TradeTracker. Передавать через kwargs в should_close
- **Альтернатива**: Сделать MacdxSignalGenerator singleton per symbol (как MarketRegimeDetector)

### Совместимость с бэктестом:
- **Проблема**: Бэктест использует тот же MacdxSignalGenerator — новая логика должна работать и в бэктесте
- **Решение**: exit_context передаётся через kwargs, бэктест может создавать свой context dict

### Обратная совместимость:
- **Проблема**: Существующие позиции не имеют exit_context
- **Решение**: Все новые проверки имеют `if not position_context: return False` — если контекста нет, работают только старые правила

## Риски и меры предосторожности

### Риски преждевременного выхода:
- **Митигация**: Фильтр ADX > 30 запрещает выход по ослаблению в сильном тренде
- **Митигация**: Минимум 2 свечи в позиции + 3 свечи после пика для ослабления
- **Тестирование**: Бэктест покажет кол-во ложных срабатываний

### Риски упущенной прибыли от позднего trailing:
- **Митигация**: Режимная адаптация — в тренде trailing шире, в боковике уже
- **Митигация**: Trailing активируется только при PnL >= 2% (значимая прибыль)

### Риски конфликтов между уровнями выхода:
- **Митигация**: Строгая приоритизация (emergency → trailing → weakening → impulse → standard)
- **Митигация**: Каждый уровень возвращает первым — нет конфликтов

### Риски увеличения сложности кода:
- **Митигация**: Каждый тип выхода — отдельный private метод
- **Митигация**: Все параметры в конфиге, не в коде
- **Митигация**: Каждый тип выхода можно отключить через `"enabled": false`

## Ожидаемые результаты

### Реалистичные метрики улучшения (на основе 1h таймфрейма):
- **Win rate**: +3-5% за счёт выхода ближе к пику и trailing profit
- **Средняя прибыль на сделку**: +10-15% за счёт оптимальных выходов на ослаблении
- **Максимальная просадка**: Снижение на 15-20% за счёт ATR trailing stop и emergency выходов
- **Время реакции**: Остаётся 60 секунд (один цикл) — MACDX не предназначен для субсекундных реакций
- **Sharpe ratio**: Ожидаемый рост за счёт более стабильных выходов

### Надёжность системы:
- **Приоритизированные выходы**: Каждый уровень с чётким приоритетом
- **Адаптивность**: Параметры подстраиваются под рыночный режим
- **Обратная совместимость**: Старые правила работают как fallback
- **Конфигурируемость**: Все пороги в JSON, каждый компонент отключаем
