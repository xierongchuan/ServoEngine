# План реализации: Оптимизация выходов из позиций в MACDX стратегии

## Цель
Реализовать точные и надёжные алгоритмы выхода из позиций для максимизации прибыли:
1. Выход в середине MACD тренда (когда импульс начинает ослабевать)
2. Быстрое закрытие на удачных пампах (секунды)

## Анализ текущей логики выхода (MACDX)

### Существующие сигналы закрытия:
- **MACD разворот**: Гистограмма меняет знак (hist < 0 для лонгов)
- **RSI экстремумы**: >80 для лонгов, <20 для шортов
- **Прибыль с RSI**: +3% прибыли + экстремальный RSI
- **Отсутствие динамики**: Только статические правила

### Проблемы текущей системы:
- Выход только при полном развороте тренда (упущенная прибыль)
- Нет быстрого закрытия на пампах
- Нет масштабирования позиций

## Предлагаемая система выходов

### 1. Выход в середине MACD тренда

#### Алгоритм ослабления импульса:
```python
# Отслеживание пика гистограммы
class MacdTracker:
    def __init__(self):
        self.peak_hist = 0
        self.peak_time = 0

    def update_peak(self, hist):
        if hist > self.peak_hist:
            self.peak_hist = hist
            self.peak_time = time.time()

    def get_weakening_ratio(self, current_hist):
        return current_hist / self.peak_hist if self.peak_hist > 0 else 1.0
```

#### Динамические уровни выхода:
- **Ранний выход** (70% от пика): Для агрессивных настроек
- **Оптимальный выход** (50% от пика): Баланс прибыли/риска
- **Поздний выход** (30% от пика): Для удержания тренда

#### Пересечение сигнальной линии:
```python
# Выход при медвежьем пересечении для лонгов

### 3. Масштабирование выхода (Scale-out)

#### Многоуровневое закрытие с учётом уже закрытых частей:

def should_emergency_close(position, realtime_data, macd_hist, prev_macd_hist):
    # Проверка: не закрывать если позиция соответствует текущему MACD тренду
    position_matches_trend = (
        (position.is_long and macd_hist > 0) or  # Лонг в бычьем тренде
        (not position.is_long and macd_hist < 0)  # Шорт в медвежьем тренде
    )

    if position_matches_trend:
        return {"should_close": False, "reason": "Position matches current MACD trend", "urgency": "low"}

    # Мгновенный MACD разворот
    if position.is_long and prev_macd_hist > 0 and macd_hist < -0.1:
        return {"should_close": True, "reason": "MACD instant reversal", "urgency": "critical"}

    # Ценовой памп против позиции
    price_change_pct = (current_price - entry_price) / entry_price * 100
    adverse_movement = (
        (position.is_long and price_change_pct < -1.0) or     # Лонг: падение >1%
        (not position.is_long and price_change_pct > 1.0)     # Шорт: рост >1%
    )
    if time_in_trade < 15 and adverse_movement:
        return {"should_close": True, "reason": "Counter-pump detected", "urgency": "critical"}

    # RSI экстремальный разворот
    if position.is_long and prev_rsi > 70 and rsi < 30:
        return {"should_close": True, "reason": "RSI extreme reversal", "urgency": "critical"}

    return {"should_close": False, "reason": "No emergency conditions", "urgency": "low"}
```

#### Анти-паника фильтры:
- **Минимальное время удержания**: Не закрывать раньше 3 секунд
- **Подтверждение тренда**: Требовать 2+ свечи подтверждения
- **Объёмный фильтр**: Игнорировать пампы без объёма

#### Экстренные выходы при развороте тренда:
- **Мгновенный разворот MACD**: Гистограмма меняет знак в противоположную сторону
- **Памп против позиции**: Цена движется против позиции >1% за 10 секунд
- **Объёмный разворот**: Объём против тренда >3x среднего
- **RSI экстремальный разворот**: RSI переходит из >70 в <30 (лонг) или наоборот
- **Защита от ложных сигналов**: HOLD если позиция соответствует текущему MACD тренду

#### Риск-менеджмент:
- **Максимальная просадка**: Закрыть если убыток > 2%
- **Временной лимит**: Закрыть через 10 минут независимо от прибыли
- **Корреляция активов**: Не держать все позиции в одном направлении

#### Проблема текущей архитектуры:

- Цена обновляется только при полном цикле

class RealtimePriceMonitor:
    def __init__(self, symbol):
        self.symbol = symbol
        self.price_buffer = deque(maxlen=50)  # Последние 50 цен (оптимизировано)
        self.volume_buffer = deque(maxlen=50)
        self.last_update = 0
        self.pump_callbacks = []

    def on_price_update(self, price, volume, timestamp):
        self.price_buffer.append((price, timestamp))
        self.volume_buffer.append((volume, timestamp))
        self.last_update = timestamp

        # Проверка на пампы в реальном времени
        if self.detect_pump():
            for callback in self.pump_callbacks:
                callback(self.symbol, "pump_detected", self.get_pump_data())

    def detect_pump(self):
        if len(self.price_buffer) < 5:
            return False

        recent_prices = [p for p, t in list(self.price_buffer)[-5:]]
        price_change = (recent_prices[-1] - recent_prices[0]) / recent_prices[0] * 100

        # Детектировать значительные движения (>1.5% за 5 обновлений)
        return abs(price_change) > 1.5

    def detect_counter_pump(self, position):
        """Детектировать памп против позиции"""
        if len(self.price_buffer) < 5:
            return False

        recent_prices = [p for p, t in list(self.price_buffer)[-5:]]
        price_change = (recent_prices[-1] - recent_prices[0]) / recent_prices[0] * 100

        # Для лонга: закрывать если цена падает >1%
        # Для шорта: закрывать если цена растёт >1%
        if position.is_long and price_change < -1.0:
            return True
        elif not position.is_long and price_change > 1.0:
            return True

        return False
```

##### 2. Отдельный цикл мониторинга пампов:
```python
class PumpMonitor:
    def __init__(self, symbol):
        self.symbol = symbol
        self.price_history = deque(maxlen=50)  # Для анализа трендов
        self.last_pump_check = 0
        self.monitoring_active = False

    def start_monitoring(self):
        """Запуск отдельного цикла мониторинга"""
        self.monitoring_active = True
        threading.Thread(target=self._monitoring_loop, daemon=True).start()

    def _monitoring_loop(self):
        """Отдельный цикл проверки пампов при каждом обновлении цены"""
        while self.monitoring_active:
            # Ждём нового обновления цены
            price_update = self.wait_for_price_update()

            if price_update:
                # Анализ на пампы при каждом обновлении
                pump_signal = self.analyze_pump_realtime(price_update)

                if pump_signal and self.should_trigger_pump_exit():
                    # Мгновенное закрытие позиции
                    self.trigger_emergency_close(pump_signal)

    def analyze_pump_realtime(self, price_update):
        """Анализ пампа на каждом ценовом обновлении"""
        self.price_history.append(price_update)

        if len(self.price_history) < 5:
            return None

        # Быстрый анализ: рост >1.5% за 5 обновлений
        recent_prices = [p['price'] for p in list(self.price_history)[-5:]]
        price_change = (recent_prices[-1] - recent_prices[0]) / recent_prices[0] * 100

        if price_change > 1.5:
            return {
                'type': 'pump',
                'strength': price_change,
                'direction': 'up' if price_change > 0 else 'down'
            }

        return None

    def should_trigger_pump_exit(self):
        """Проверка условий для срабатывания"""
        # Проверяем, есть ли открытая позиция против пампа
        if not self.has_opposite_position():
            return False

        # Проверяем, что позиция не соответствует тренду
        if self.position_matches_current_trend():
            return False

        return True

### Интеграция с MACDX стратегией с приоритетами:
```python
def should_close(self, analysis, position, **kwargs):
    """
    Приоритизация выходов:
    1. Экстренные выходы (критично) - переопределяют всё
    2. Scale-out (высокий приоритет) - для прибыльных позиций
    3. Быстрые пампы (высокий приоритет) - для внезапной прибыли
    4. Обычные MACD выходы (нормальный приоритет)
    """

    realtime_data = kwargs.get('realtime_monitor', None)
    pnl_pct = position.get_pnl_pct()

    # 1. ЭКСТРЕННЫЕ ВЫХОДЫ - максимальный приоритет
    if realtime_data and realtime_data.detect_emergency_exit(position):
        return {"should_close": True, "reason": "Emergency exit", "urgency": "critical"}

    # 2. SCALE-OUT - для прибыльных позиций
    if pnl_pct > 0:
        scale_out_signal = self._check_scale_out_conditions(position, analysis)
        if scale_out_signal:
            return scale_out_signal  # Может быть частичным закрытием

    # 3. БЫСТРЫЕ ПАМПЫ - внезапная прибыль
    if realtime_data and realtime_data.detect_quick_profit_pump(position):
        return {"should_close": True, "reason": "Quick profit pump", "urgency": "high"}

    # 4. ОБЫЧНЫЕ MACD ВЫХОДЫ - базовая логика
    return self._check_macd_exit_conditions(analysis, position)
```
